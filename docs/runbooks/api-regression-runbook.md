# API Regression Runbook

## Purpose
Define the recurring API regression process for banking API releases.

## Steps
1. Confirm scope and impacted APIs
2. Validate environment readiness
3. Load test data
4. Execute ReadyAPI/Postman/REST Assured suites
5. Run SQL reconciliation checks
6. Review failed tests and defects
7. Publish Allure/HTML reports
8. Update release dashboard
9. Provide Go/No-Go recommendation
